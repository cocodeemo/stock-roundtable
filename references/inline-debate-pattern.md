# 内联辩论执行模式

当无法使用脚本模式（demo.py 需要 API key 或 API key 不在当前 shell 环境）时，Agent 自身扮演所有辩论角色，直接在对话中输出结构化辩论。

## 执行模板

### Round 1：独立陈述（每个角色 3-5 个要点）

每个角色以 `### 🟢/🔴/🔵/🟡/⚫ 角色名 发言` 开头，要点用编号列表。保持角色语气一致。

### Round 2：互相引用反驳

每个角色必须用 `**@角色名 你的XX论点**` 格式引用对方具体论点后反驳或补充。这是辩论质量的核心——**禁止空泛回应**。

质疑者（⚫）在 Round 1 负责审视所有首轮论点的逻辑漏洞，在 Round 2 继续指出辩论中共同回避的问题。

### 裁判最终报告

以结构化 Markdown 表格输出：
- **共识项**（带共识度百分比）
- **分歧项**（对比不同立场）
- **风险评估**（概率 + 影响等级）
- **综合建议**（带置信度百分比 + 具体操作方案）
- **关键监控指标**

## 金融/投资分析场景的角色适配

| 标准角色 | 金融场景映射 | 重点 |
|---------|------------|------|
| 乐观派 | 多头方 | 寻找催化剂、估值修复逻辑、行业拐点信号 |
| 悲观派 | 空头方 | 识别下行风险、质疑估值、找财务漏洞 |
| 技术派 | 行业分析师 | 供需平衡表、成本曲线、产能周期、技术替代 |
| 质疑者 | 风控/二阶审查 | 审核推理过程、指出回避的问题、量化缺失 |

注意：金融场景下"业务派"通常与"技术派"合并为"行业分析师"，因为行业供需分析已经覆盖了业务逻辑。

## 本次会话的实战案例

**辩题**：锂矿持仓决策——西藏矿业(000762) + 中矿资源(002738)

**角色**：乐观派 / 悲观派 / 行业技术派 / 质疑者

**核心产出**：
- 共识：锂价低位、短期承压、西藏矿业有成本优势
- 分歧：是否已见底（乐观 vs 悲观对产能周期的不同判断）
- 质疑者贡献：指出三方回避了"仓位管理量化方案"和"替代标的"问题
- 裁判建议：西藏矿业保留底仓 ≤50%，中矿资源减仓 ≥50%

## 与 WorkBuddy 圆桌会议的差异

| 维度 | WorkBuddy | Hermes 内联辩论 |
|------|----------|----------------|
| 角色 | 预设金融分析师 | 完全可定制 |
| 轮次 | 单轮并行发言 | 2-3 轮互相引用反驳 |
| 输出 | Web 对话流 | 结构化报告（共识/风险/置信度/建议） |
| 底层 | WorkBuddy 模型 | 用户自己的模型 |

## Pitfalls

- **角色区分度**：内联模式下所有角色由同一个 Agent 扮演，容易出现"自己附和自己"的假辩论。解决：Round 2 必须硬性要求引用 @角色 具体论点，不能只泛泛同意
- **质疑者时机**：质疑者在 Round 1 审视逻辑漏洞，Round 2 指出回避问题，两轮职责不同
- **不给出量化方案是无效辩论**：裁判必须给出具体操作建议（保留 X%、减仓 Y%），不能只说"持有"或"卖出"
- **实际行情数据比记忆可靠**：尽量用实时 API 数据（东方财富等）来锚定论点，避免纯凭训练数据记忆
