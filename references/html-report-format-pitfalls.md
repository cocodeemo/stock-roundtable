# HTML 报告格式 Pitfalls — Round 2 专用

记录了 v2.0.0 → v2.0.1 修复的 Round 2 LLM 输出格式兼容性问题。

## 问题 1：评分表格列头识别过窄

**旧行为**：`html_report.py:554` 只匹配 `分数|评分|分值|得分` 四种列头。

**问题**：Round 2 表格用了 `旧分`、`新分`、`调整` 等非标准列头 → 表格不添加 `score-table` class → 无样式渲染。

**修复**：扩展为 `分数|评分|分值|得分|旧分|新分|调整`。

---

## 问题 2：`_add_bar_to_row` 无法处理尾随文本

**旧正则**：
```python
score_match = re.search(r'<td[^>]*>(?:<strong>)?([\d.]+)(?:</strong>)?</td>', row)
```
要求 `</td>` 紧跟在数字或 `</strong>` 后面。

**问题**：Round 2 输出 `**30** (下调10分)` → 数字后跟尾随文本，正则不匹配 → 该行无进度条。

**修复**：
```python
m = re.search(r'<td[^>]*>(?:<strong>)?([\d.]+)(?:</strong>)?(.*?)</td>', row)
trailing = m.group(2)
return row[:m.start()] + '<td>' + bar_html + trailing + '</td>' + row[m.end():]
```

## 问题 3：对比表分数提取——表格兜底误取维度分

**旧行为**：当 4 列表格（`维度|旧分|新分|理由`）存在时，取 `cells[-2]` 作为分数。

**问题**：笨韭 Round 2 表格末行 `估值水位 | 5 | 5 | 理由...` → 取到 `5`（估值水位维度分），覆盖 Round 1 的实际总分 `65.75`。

**修复**：表格兜底只取标准 3 列表格（`len(cells) == 3`，`cells[1]` = 分数列）。

---

## 问题 4：评分提取正则遗漏格式

**旧行为**：不支持以下格式：
- `总分 **40/100**`（空格非冒号）
- `综合评分：30/100`、`总评：XX/100`
- `XX/100，结论`（尾随逗号非 `分` 或 `→`）

**新增正则**：
```python
# 1b. Markdown bold，空格非冒号
r'总分\s+\*?\*?([\d.]+)\*?\*?\s*/\s*100'

# 1c. 综合评分/总评
r'(?:综合评分|总评|综合得分)[：:]\s*\*?\*?([\d.]+)\*?\*?\s*/\s*100'

# 2. 去掉 (?:分|→) 尾随限制
r'(?<!\d)([\d.]+)\s*/\s*100'
```
