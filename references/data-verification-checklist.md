# 股票数据校验清单

每次拉完行情+财报数据后，必须逐项校验。**数据错误比分析错误更致命**——分析错了可以辩，数据错了整份报告废掉。

## 必须校验项

### 1. 送转股除权调整

- 调用 `stock_dividend_cninfo(symbol)` 获取除权记录
- 找出所有「除权日 > 最新报告期」的送转股
- 计算累计调整系数：`1 + (送股比例 + 转增比例) / 10`
- 对以下字段做除法调整：EPS、每股净资产、每股经营现金流
- **NaN 陷阱**：`stock_dividend_cninfo()` 无送股时返回 NaN，必须用 `x == x` 检测
- 漏调整后果：臻镭 10转增4，EPS 0.18→应 0.13，PE 算成 97x→实际 136x

### 2. 52 周高低（必须用前复权 K 线）

- ❌ 禁止：直接用腾讯 `qt.gtimg.cn` 的 `fields[47]/[48]`——这些值不除权
- ✅ 必须：拉前复权 K 线 `fqkline/get?param=sh688270,day,,,250,qfq`
- 取最近 250 根 K 线的真实 `max(high)` 和 `min(low)`
- 漏校验后果：API返回 52高=82/低=55 → 当前 70 距高点 14.6% → 莫大误判"顶部"。实际前复权高=163/低=30 → 跌57% → 实为中位。**所有角色的股价位置分数全部反了**

### 3. PE 交叉验证（v1.9.1+ 三方）

- 腾讯 PE：`qt.gtimg.cn` fields[39]
- 东财 PE：`push2.eastmoney.com` f162（÷100）
- 计算 PE：`price / (EPS × 4)`（Q1年化）
- 三源最大偏差 > 15% → 报警
- 偏差合理 → 打印通过信息
- 东财财务（营收/净利/毛利率）是年度口径，AKShare 是季度口径，**不能直接对比**

### 4. 市值交叉验证

- 腾讯市值：fields[45]（亿）
- 东财市值：f116（元→÷1e8 转亿）
- 偏差 > 10% → 报警
- 华特实测：腾讯 322亿 vs 东财 322亿（完全一致 ✅）

### 5. 腾讯 API 字段索引

极易出错，必须严格按此索引：
```
fields[39] = 动态PE(TTM)     ← 不是 [50]
fields[45] = 总市值(亿)       ← 不是 [46]（[46]是流通市值）
fields[47] = 52周最高         ← 不是 [44]
fields[48] = 52周最低         ← 不是 [45]
```
踩坑：曾把 52 周高=低=210 显示在报告上，市值 9 亿而非 210 亿。

### 6. AKShare 数值解析

`stock_financial_abstract_ths()` 返回中文单位字符串（"1.33亿"、"6510.73万"），必须用 `_parse()` 转换。同比字段 "%" 需去符号。

### 7. 极端估值预警

- PE > 200 → 报警
- PS（市值/营收）> 100 → 报警
- 总市值 < 10 亿 → 疑为数据错误（臻镭曾误显示 9 亿）

### 8. ST/特殊状态

- 腾讯 API 名称字段包含 "ST" 或 "*ST" → 必须在问题中显式标注 ⚠️
- 不标注 → LLM 按正常经营假设辩论 → 整场失效

## 校验输出格式

校验通过：`✅ 数据校验通过`
校验失败：`❌ 数据异常: PE偏差XX%（计算X vs API Y）; PS Zx 极高`

校验失败时仍然继续跑辩论，但在问题中附加警告标记。
